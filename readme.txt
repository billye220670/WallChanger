项目背景：允许用户上传室内图片，拖拽材质到对应墙面换材质（通过ai洗图）最终得到一个崭新风格的图片。

项目平台：webapp项目，主要适配移动端竖屏体验。之后要部署到vercel，允许手机远端访问。

技术栈：架构和技术栈你来选择，要快速迭代。比如vite。


详细步骤：
1）用户上传室内空间照片，上传后自动在后台开始run下面的task：

	1-1）调用flux4b api洗图，提示词为“remove all object, keep only hard finishes, keep structure unchanged.” 尺寸参数保持与原图一致。

	1-2）将步骤1-1的结果图调用SAM3识别成彩色蒙版，尺寸参数保持与步骤2结果一致。

	1-3）将步骤1-2彩色蒙版做高斯模糊处理，模糊值你来定，不要太大。这一步是为了下步洗图准备。

	1-4）将步骤1-3模糊处理后的结果图调用flux4b api洗图，提示词：“refine the mask, the connection between different color should have no gap” 尺寸参数保持与蒙版图一致。


2）界面显示识别完成。这时app中：用户看到的还是原图，但是后台我们已经有了refine过后的彩色蒙版图。

	2-1）用户从底部拖出抽屉bottomsheet，展现出一个材质库。用户可以任意拖动材质库中的材质到原图上。拖动时会根据蒙版信息，把对应的墙面区域边缘高亮提示用户。

	2-2）一旦用户刚刚拖拽的材质球图片在一个区域松开，则run下面的task：
		
		2-2-1）对应的墙面区域开始有骨架屏加载那种瀑布刷动的效果，意为正在生成中。

 		2-2-1）调用flux4b api洗图，提示词为："based on image 2, change all wall material in image 1." 尺寸参数与之前保持一致。这一步需要传两个图，按顺序分别是原图和材质球参考图（即材料库中被拖拽上的图）
	
		2-2-2）生成完后，把2-2-1的结果图叠加在原图上并apply对应区域的mask。届时瀑布刷动的效果消失。变为结果。

		2-2-3）依次类推其他区域。如果拖拽到已有结果的区域则复写。

3）用户确认无需再有需要更替的墙面材质后，点击底部的一键焕色按钮。屏幕变暗以及不可操作。有一个魔法效果。和一个圆环加载。然后run下面task：

	3-1）此时屏幕上已有合成好的最终图，分别由原图叠加不同mask区域的洗图结果而成。用这张最终的合成图：

		3-1-1）对合成图进行模糊处理，和1-3类似。

		3-1-2）对模糊后的合成图调用flux4b API洗图，提示词为"realistic render",尺寸参数仍然保持一致。

		3-1-3）洗完后得到最终结果，弹出一个底部抽屉把图片展示给用户。并提供保存选项。
